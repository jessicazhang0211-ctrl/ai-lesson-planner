# AI Lesson Planner 毕业论文写作大纲与项目资料

更新时间：2026-03-15
适用项目：基于 Flask + MySQL + 原生前端的教学辅助系统（教师端 + 学生端）

## 1. 论文可直接使用的总大纲

### 第一章 绪论
1. 研究背景与意义
- 中小学教师备课、出题、批改、学情追踪任务重。
- 大模型技术可用于教案生成、题目生成、学习分析，提升教学效率。
- 本项目面向真实教学流程，打通“生成-发布-作答-批改-统计-分析”闭环。

2. 国内外研究现状
- 智能教学系统：多聚焦单点能力（仅题库或仅推荐）。
- 大模型教育应用：质量高但可控性、结构化输出和落地流程仍是难点。
- 本系统价值：结合规则约束与业务流程，强调可操作与可落地。

3. 研究内容与目标
- 设计并实现教师端与学生端协同系统。
- 引入 AI 服务完成教案与习题生成。
- 建立班级、作业、成绩、统计、学情分析模块。
- 形成可部署、可演示、可扩展的工程实现。

4. 论文组织结构
- 绪论、需求分析、系统设计、系统实现、测试与分析、总结与展望。

### 第二章 需求分析
1. 角色与业务需求
- 教师：注册登录、班级管理、学生导入、教案生成、习题生成、发布任务、批改复核、统计分析。
- 学生：学号登录、查看任务、保存草稿、提交作业、查看成绩、查看评语与学情建议。

2. 功能需求
- 账号认证与权限区分（教师邮箱登录、学生学号登录，学生禁用自助注册）。
- 班级与学生管理（导入/导出、自动生成学号、自动创建学生账号）。
- 教案生成与历史管理（可编辑、可删除）。
- 习题生成与结构化存储（JSON 题目结构）。
- 资源发布（按班级、按学生下发）。
- 学生提交与自动判分（客观题自动判分，主观题待教师评分）。
- 教师批改与成绩汇总。
- 多维统计（班级总览、资源统计、学生个人分析）。

3. 非功能需求
- 易用性：前后端分离，静态页面部署简单。
- 可维护性：模块化蓝图、模型分层、统一响应格式。
- 安全性：JWT 鉴权、密码哈希存储。
- 扩展性：AI 服务单独封装，可替换模型。

### 第三章 系统设计
1. 总体架构设计
- 架构类型：前后端分离。
- 前端：原生 HTML/CSS/JavaScript，分教师端与学生端页面。
- 后端：Flask + SQLAlchemy + Blueprint 路由模块。
- 数据层：MySQL。
- AI 能力：Google Gemini（经统一 AIService 调用）。

2. 模块划分设计
- 认证模块：注册、登录、令牌签发与校验。
- 用户模块：个人资料与密码修改。
- 班级模块：班级 CRUD、学生导入导出、统计总览。
- 教案模块：AI 生成、格式清洗、历史记录。
- 习题模块：AI 生成、结构化解析、历史管理。
- 资源模块：发布、撤回、评阅、资源统计。
- 学生模块：作业领取、保存、提交、复盘、学情分析。

3. 数据库设计
- users：系统账号表（教师账号 + 学生映射账号）。
- classes：班级表。
- class_students：班级学生表。
- student_profiles：学生账号与班级学生实体映射表。
- lessons：教案表。
- exercises：习题表（含结构化 content_json）。
- resource_publications：资源发布表。
- resource_assignments：任务分发表。
- exercise_submissions：学生作答与批改表。

4. 关键业务流程
- 教师流程：登录 -> 创建班级 -> 导入学生 -> 生成教案/习题 -> 发布 -> 查看统计/评阅。
- 学生流程：登录 -> 查看任务 -> 保存草稿 -> 提交作答 -> 查看成绩与分析。
- 评分流程：自动评分（客观题）+ 人工评分（主观题）-> 总分写回任务。

### 第四章 系统实现
1. 后端实现
- Flask 工厂模式 create_app。
- Blueprint 分模块注册。
- SQLAlchemy 模型与关系。
- 统一响应与异常处理中间件。
- JWT 鉴权装饰器。

2. 前端实现
- 教师端页面：班级、教案、习题、资源发布、评阅、设置、仪表盘。
- 学生端页面：首页、任务、作答、复盘、成绩、设置。
- i18n 支持（中英文文案字典化，后端部分接口支持 lang 参数）。

3. AI 能力实现
- 教案生成：严格格式提示词 + 结果清洗 sanitize，保证可打印结构。
- 习题生成：要求输出 JSON，后端提取结构化字段。
- 学情分析：规则分析为基础，条件允许时调用 AI 生成诊断建议。

4. 关键实现细节
- 学生导入支持 xlsx/xls/csv/json，多表头兼容（中英文）。
- 学号自动规则：YYYY0CCNNN。
- 导入后自动创建学生账号并绑定 student_profile，默认密码 123456。
- 资源发布后批量生成 assignment 记录，便于追踪完成状态。

### 第五章 测试与结果分析
1. 测试策略
- 功能测试：按角色与业务流程逐条验证。
- 接口测试：注册登录、班级、生成、发布、提交、评阅、统计。
- 异常测试：缺参数、权限不匹配、重复账号、无效文件格式。

2. 可展示测试结果
- 登录成功率、接口响应正确率。
- 作业提交流程的状态迁移正确性（assigned -> saved -> completed）。
- 自动评分与人工评分叠加正确性。

3. 性能与稳定性讨论（可选）
- AI 调用时延。
- 批量导入数据时的处理耗时。
- 并发场景下的数据库事务一致性风险。

### 第六章 总结与展望
1. 工作总结
- 已实现教学业务闭环。
- 具备 AI 增强的实际应用价值。

2. 不足与改进
- 当前使用 db.create_all，生产建议迁移到数据库版本管理。
- 权限模型可进一步细化（教师助教、多班共享）。
- 可加入题目知识点标签与个性化推荐。
- 可增加日志审计、限流与缓存机制。

## 4. 数据库表设计说明（论文可作为表结构章节草稿）

1. users
- 主键：id
- 核心字段：name, email, password_hash
- 扩展字段：nickname, gender, bio, phone, school, major, job_title, avatar_url
- 说明：教师账号直接使用；学生通过 student_profiles 映射。

2. classes
- 主键：id
- 外键：created_by -> users.id
- 字段：name, description, status, code, created_at, stage, allow_join, note

3. class_students
- 主键：id
- 外键：class_id -> classes.id
- 字段：name, stu_id, status, parent_phone, accuracy, submit, created_at

4. student_profiles
- 主键：id
- 关联：user_id(唯一), class_id, student_id
- 字段：analysis_json, analysis_updated_at, analysis_latest_completed_at
- 说明：完成学生登录账号与班级学生实体的桥接。

5. lessons
- 主键：id
- 外键：created_by -> users.id
- 字段：title, description, created_at

6. exercises
- 主键：id
- 外键：created_by -> users.id
- 字段：title, description, content_json, created_at

7. resource_publications
- 主键：id
- 字段：resource_type, resource_id, class_id, student_ids, accuracy_rule, mode, revoked, revoked_at, created_by, created_at
- 说明：存储“发布动作”的事实记录。

8. resource_assignments
- 主键：id
- 外键：publish_id -> resource_publications.id, student_id -> class_students.id
- 字段：status, score, completed_at, created_at

9. exercise_submissions
- 主键：id
- 外键：publish_id, student_id
- 字段：answers, auto_result, auto_score, teacher_score, teacher_detail, total_score, teacher_comment, status, created_at, updated_at

## 6. 可写“创新点/特色点”建议

1. AI 与规则结合的可控生成
- 教案并非仅生成文本，而是通过格式约束 + 清洗函数确保可直接使用。
- 习题输出要求 JSON 结构，便于自动判分与统计。

2. 教学业务闭环完整
- 从“资源生产”到“学习结果反馈”完整贯通，适合教育场景落地。

3. 双角色协同
- 教师与学生端业务边界清晰，既能管理全班又能追踪个体。

4. 自动化运维能力
- 导入学生时自动建账号、自动学号生成，降低教师管理成本。

5. 学情分析双引擎
- 规则分析兜底，AI 分析增强，兼顾可解释性与智能化。

## 7. 可写“难点与解决方案”模板

1. 难点：AI 输出不稳定
- 方案：在提示词中强约束结构，并在后端进行 sanitize 与 JSON 提取。

2. 难点：多格式导入兼容
- 方案：支持 xlsx/xls/csv/json，增加表头归一化与 header 行自动识别。

3. 难点：角色认证与权限
- 方案：JWT 鉴权 + 资源归属校验 + 学生 profile 映射。

4. 难点：评分流程一致性
- 方案：自动评分写入 submission，人工评分叠加后回写 assignment。
