# GDPR 与英国伦理整改验收清单（可执行版）

更新时间：2026-03-15
用途：用于项目整改跟踪、论文附录、答辩展示
说明：本清单为工程实践与学术项目常用控制项，不构成法律意见

## 使用方式

1. 每完成一项就把状态从 `未开始` 改为 `已完成`。
2. 每项必须补齐“证据材料”，否则视为未完成。
3. 建议先完成 P0，再进入 P1/P2。

| 编号 | 事项 | 验收标准 | 证据材料 | 状态 |
|---|---|---|---|---|
| P0-1 | 发布隐私政策与数据处理说明 | 有面向教师/学生/监护人的隐私说明页面，明确收集字段、用途、保存时间、第三方处理 | 隐私政策文档链接 + 页面截图 | 未开始 |
| P0-2 | 明确处理合法性基础（lawful basis） | 每类处理活动都有合法性基础映射（如合同、公共任务、合法利益、同意） | 合法性基础对照表（文档） | 未开始 |
| P0-3 | 增加数据主体权利入口 | 至少支持：访问、导出、删除、更正申请流程 | API 文档 + 接口测试记录 | 未开始 |
| P0-4 | 删除默认弱口令机制 | 学生首次登录必须强制改密；不再长期使用固定初始密码 | 代码改动 + 测试报告 | 已完成 |
| P0-5 | 生产环境安全基线 | 强制 HTTPS、限制 CORS 白名单、关闭默认开发密钥 | 配置文件截图 + 安全测试记录 | 部分完成 |

## 验收打分（建议用于阶段汇报）

1. 完成率计算：
- 完成率 = 已完成项数 / 总项数 × 100%

2. 阶段门槛建议：
- < 40%：仅可内部开发测试，不建议对外试点
- 40% - 79%：可小范围受控试点，需签署告知
- >= 80%：可进入正式伦理申请/合规评审阶段

## 本轮已落地改动（2026-03-15）

1. 已新增密码强度校验（注册与改密均要求：至少 8 位，含大小写、数字、特殊字符）。
2. 已新增学生默认密码门禁：学生若仍为默认密码，仅允许先改密。
3. 已在登录返回中增加 `must_change_password` 标志位，便于前端引导。
4. 已将 CORS 改为白名单配置（可通过环境变量 `CORS_ORIGINS` 维护）。
5. 自动化测试已覆盖“学生首次登录被拦截 -> 改密 -> 恢复访问”全流程。
